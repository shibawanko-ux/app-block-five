## 1. プロダクト概要

### 1.1 コンセプト

物理演算付き5目並べゲーム。スイカゲームのように自由な位置からブロックを落とし、物理的に積み上がった状態で5つ揃えると勝利する新感覚対戦ゲーム。

### 1.2 ターゲット

- たくやさん個人での遊び・友人との対戦
- 将来的にネイティブアプリ化を見据えたWeb MVP

---

## 2. ゲームルール

### 2.1 基本ルール

| 項目 | 内容 |
|------|------|
| **盤面** | グリッド（列 × 行）。物理演算の結果をグリッドにスナップして管理 |
| **操作** | 上部でマウスを左右スライドし、好きな位置でクリックして落とす（スイカゲーム式） |
| **ブロック形状** | 正方形 |
| **物理挙動** | 重力で落下・他のブロック・壁に衝突して積み上がる（Matter.js） |
| **スナップ** | ブロックが静止したら最も近いグリッドセルに自動スナップ |
| **勝利条件** | 縦・横・斜め（右上/右下）のいずれかで自分の色が `winLength` 個連続する |
| **引き分け条件** | 盤面の最上段がすべて埋まった状態 |
| **手番** | プレイヤー1 → プレイヤー2（またはCPU）→ 繰り返し |

### 2.2 勝利判定の方向

- ➡ 横方向（水平）
- ⬇ 縦方向（垂直）
- ↘ 斜め（右下）
- ↗ 斜め（右上）

### 2.3 物理スナップの仕様

1. ブロックの速度が閾値（0.4px/f）以下かつグリッド内に入ったフレームを計測
2. 18フレーム（約0.3秒）継続したら「静止」と判定
3. ブロックのX座標から列を決定（`Math.round((x - CELL/2) / CELL)`）
4. その列の最下段の空きセルに配置（物理Y位置は無視）
5. 物理ボディを削除し、グリッド上のセルとして記録

## 3. 機能要件

### 3.1 モード

| モード | 説明 |
| -------- | ---- |
| **2人対戦** | 同一デバイスで2人が交互に操作 |
| **CPU対戦** | プレイヤー1 vs CPU（ミニマックス AI） |
| **オンライン対戦** | Firebase Realtime Database 経由で遠隔2人対戦（詳細は別ドキュメント参照） |

### 3.2 ゲームフロー

```
モード選択画面
  └─ [2人対戦 / CPU対戦] 選択
      └─ ゲーム画面（Canvas描画）
          ├─ プレビュー表示（上部・マウスに追従）
          ├─ クリックでブロック落下（物理演算）
          ├─ 静止後にグリッドスナップ
          ├─ 勝敗・引き分け判定
          └─ [もう一回 / モード選択へ] ボタン
```

### 3.3 UI要件

| 要素 | 仕様 |
|------|------|
| 描画エンジン | HTML5 Canvas（60fps requestAnimationFrame） |
| プレビュー | 落下前にマウス列に半透明ブロックを表示 + 破線の縦線 |
| プレイヤー識別 | P1 = 青（#3b82f6）、P2/CPU = 赤（#ef4444） |
| ターン表示 | React UI overlay で色付き表示。カウントダウン秒数テキストのみ（バーなし） |
| 勝利演出 | 揃った5マスを金色ハイライト |
| CPUターン | 0.7秒後に自動でブロックを落とす |

> **タイマーバー（進捗バー）は廃止**：CSS `transition: width 1s linear` による画面のカクつきが発生するため削除。秒数テキストのみで十分。

### 3.4 CPU仕様（MVP）

| 項目 | 内容 |
| ---- | ---- |
| アルゴリズム | ミニマックス法 + α-β 枝刈り |
| 探索深さ | 4手先 |
| 入力 | グリッドスナップ後の仮想ボードを評価（物理位置は不要） |
| 出力 | 列番号（0〜COLS-1）→ その列の中央X座標からブロック生成 |

### 3.5 爆弾アイテム仕様

各プレイヤーはゲームを通じて1回だけ爆弾を使用できる。

| 項目 | 仕様 |
| ---- | ---- |
| 使用回数 | 1ゲームにつき各プレイヤー1回（`BOMBS_PER_PLAYER = 1`） |
| 設置対象 | **相手プレイヤーの**ブロックが置かれているセルのみ |
| ターン消費 | 設置した時点でそのターンを消費（ブロック落下は行わない） |
| 爆発タイミング | 設置から**2ターン後**（ターン終了ごとにカウントダウン） |
| 爆発範囲 | 設置セルを中心とした **十字形5セル（中心＋上下左右）** ※斜め4セルは含まない |
| 重力処理 | 爆発後、上に乗っていたブロックは重力で下へ落下する |
| UI | 爆弾ボタンをクリックして爆弾モードに切り替え → 対象セルをクリックで設置。カウントダウンバッジ（💣）をブロック上に表示 |

#### 爆発アニメーション

| フェーズ | フレーム | 内容 |
| -------- | -------- | ---- |
| フラッシュ | 0〜21f（0.35秒） | 爆心地から橙→赤のラジアルグラデーションが拡散してフェードアウト |
| セル消滅 | 中心=2f・隣接=10f・角=18f 起点・各6fでフェード | 中心→上下左右→斜め角の順に順番に消える |
| 落下 | 28〜52f（0.4秒） | 上に積まれていたブロックが ease-in（重力感）で新位置へ移動 |

アニメーション中はクリック・タイマー・ブロック落下をすべてロックし、終了後にゲームを再開する。

### 3.6 設定画面

メニュー画面およびオンライン待機画面（ホストのみ）から `⚙️ Settings` ボタンで遷移できる専用画面。

| 設定項目 | デフォルト | 内容 |
| -------- | ---------- | ---- |
| **格子柄** | **OFF** | グリッドライン（盤面の縦横線）の表示/非表示を切り替える |

- `← Back` ボタンで元の画面に戻る
- 設定はリアルタイム反映（ゲームループの次フレームから即座に適用）
- ゲスト（P2）は設定画面にアクセスできない（オンラインのWaiting画面はホストのみ表示）

## 4. 盤面設定（GameConfig）

| 設定値 | MVP値 | 変更方針 |
|--------|-------|---------|
| `cols`（列数） | 10 | 将来的にUI上で選択可能にする（定数切り出し済み） |
| `rows`（行数） | 10 | 同上 |
| `winLength`（勝利条件） | 5 | 同上 |
| `CELL`（セルサイズ px） | 56 | 解像度・デバイスに応じて調整 |

## 5. 技術スタック

| レイヤー | 技術 | 備考 |
| ------- | ---- | ---- |
| フロントエンド | React 19 + TypeScript | UI overlay のみ React |
| 描画 | HTML5 Canvas | ゲーム本体の描画すべて |
| **物理エンジン** | **Matter.js** | 重力・衝突・壁 |
| ビルドツール | Vite 8 | |
| スタイル | Tailwind CSS v4 → インラインスタイル | Canvas UI はインラインスタイル |
| ゲームロジック | 自前（gameLogic.ts / cpuPlayer.ts） | グリッドベースで変更なし |
| デプロイ先（予定） | Vercel → Electron（後日） | |

## 7. スコープ外（将来対応）

| 機能 | 優先度 | 備考 |
|------|--------|------|
| 盤面サイズ変更UI | 高 | `GameConfig` 切り出し済み |
| CPU強さ選択 | 中 | `getCpuMove(depth)` の depth を渡すだけ |
| ブロック落下アニメーション強化 | 低 | 物理演算で自然な動きは確保済み |
| 対戦スコア記録 | 低 | セッション内累計 |
| タッチ操作（スマホ） | 中 | ✅ 実装済み（touchstart/move/end 対応） |
| Electron化 | 将来 | Web MVPが安定してから |
| iOS/Android化 | 将来 | React Native or Swift |
